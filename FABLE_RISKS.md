# FABLE_RISKS.md

> Ranked risk register. Severity: Critical / High / Medium / Low.
> Sources: Loop 1 (context intake) + Loop 2 (architecture reverse-engineering), 2026-07-06.
> Facts vs assumptions are marked. Items are *risks*, not confirmed vulnerabilities, until an Audit/Security Loop verifies them.

| # | Risk | Severity | Type | Evidence | Status |
|---|---|---|---|---|---|
| R-01 | **No authentication anywhere** — every API route (Docker orchestration, settings, downloads, chat) is unauthenticated by design. Any device on the LAN has full control, including the Docker daemon. | High (by-design, documented) | Security | **Verified:** README §About Security; `admin/start/kernel.ts:48` (`router.named({})`), session middleware commented out, no user tables in migrations | Accepted upstream by design; re-evaluate if exposure model ever changes |
| R-02 | **Command Center controls the Docker daemon** — `/var/run/docker.sock` mounted into admin, updater, and dozzle containers (`install/management_compose.yaml`). Combined with R-01, reaching port 8080 is effectively root-on-host. | High | Security | **Verified:** `admin/app/services/docker_service.ts:30`; compose mounts confirmed | Confirmed in Loop 2 |
| R-03 | **No automated tests** — `admin/tests/` has only `bootstrap.ts`; root test script is a stub. Every change ships on typecheck/lint/manual QA only. | High | Quality | `admin/tests/`, `package.json` | Open — candidate for QA Loop |
| R-04 | **CSRF disabled + wildcard CORS + no session auth** — state-changing endpoints (install containers, delete content, self-update) are callable cross-origin: a malicious website visited by any browser on the LAN can drive `http://device:8080` (DNS-rebinding / drive-by). | High | Security | **Verified:** `admin/config/shield.ts:19` (`csrf.enabled: false // TODO`), `admin/config/cors.ts` (`origin: ['*']`, `credentials: true`), no auth middleware in `admin/start/kernel.ts` | Confirmed in Loop 2; fix candidate for Security Loop |
| R-05 | Shell/exec surface — **largely defused on inspection**: `child_process.exec` limited to fixed `lspci` (`docker_service.ts:706,721`); `nvidia-smi` via dockerode argv arrays; container `Cmd` split on spaces (not shell-executed, but naive splitting mishandles quoted args, `docker_service.ts:516`). Installer scripts still run as root. | Low–Medium | Security | Verified in Loop 2 survey | Downgraded from Medium; re-check on new exec call sites |
| R-06 | **Fork drift**: this fork's only divergence from upstream is adding `remotion` deps at repo root (PR #2) with no consuming code — unclear intent; dead deps add install weight and attack surface. | Medium | Product/Tech | Root `package.json`, commit `04a3821` | Needs human clarification |
